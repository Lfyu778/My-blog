## 一、先建立全局图景：这个博客到底是什么？

### 1.1 一句话概括

这不是手写一堆 `.html` 页面的传统网站，而是：

**用 Markdown 写文章 → 用 VitePress 自动生成网站 → 部署到 Vercel 上线**

你写的是 `.md` 文件，浏览器看到的是完整的 HTML 页面。中间那层「把 Markdown 变成网站」的工具，就是 **VitePress**。

### 1.2 和你熟悉的 HTML 对比

| 传统手写网站 | 这份博客 |
|---|---|
| 每个页面写一个 `.html` | 每篇文章写一个 `.md` |
| 导航、侧边栏每个页面复制一遍 | 在配置文件里写一次，所有页面自动带上 |
| 自己管样式、布局、响应式 | 主题已经做好，你只改颜色和细节 |
| 改完要自己上传服务器 | `git push` 后 Vercel 自动部署 |

### 1.3 项目结构（先认目录，不急着读代码）

```
my-blog/
├── package.json              ← 项目说明书：依赖什么、怎么启动

├── vercel.json               ← 告诉 Vercel 怎么构建和部署

│
├── docs/                     ← ⭐ 真正的内容都在这里
│   ├── index.md              ← 首页
```

## 三、第 1 层：立刻能上手

### 3.1 Markdown —— 写文章的语言

**是什么**
一种用纯文本写格式的轻量标记语言。`#` 是标题，`**粗体**`，`-` 是列表。VitePress 会把它转成 HTML。

**在本项目哪里**
所有文章和页面：`docs/posts/**/*.md`、`about.md`、`guestbook.md` 等。

**你要学到什么程度**
会写标题、段落、列表、链接、代码块、引用就够了。进阶再学表格和 VitePress 扩展语法（如 `::: tip`）。

**项目中的例子**（`docs/guestbook.md`）：

```markdown
---
title: 留言板
---

# 💬 留言板

欢迎在这里留下你的想法、建议、或者任何问题。

::: tip 如何留言
点击下方评论区，使用 GitHub 账号登录即可留言。
:::

<GiscusComment />
```

注意三件事：
1. 文件最上面 `---` 包起来的叫 **frontmatter**（见 4.2）
2. `::: tip` 是 VitePress 扩展，不是标准 Markdown
3. `<GiscusComment />` 是 Vue 组件，可以写在 Markdown 里（见 5.3）

**推荐练习**：打开任意一篇 `docs/posts/` 下的文章，对照渲染结果看每个符号对应什么效果。

### 3.2 Node.js + npm —— 项目的「运行时」和「包管理器」

**是什么**
- **Node.js**：让 JavaScript 能在电脑上跑（不只在浏览器里）
- **npm**：Node 的包管理器，负责安装别人写好的工具（比如 VitePress）

**在本项目哪里**
`package.json` 就是项目的「说明书」：

```json
{
  "name": "my-blog",
  "type": "module",
  "scripts": {
    "dev": "vitepress dev docs",
    "build": "vitepress build docs",
    "preview": "vitepress preview docs"
  },
  "devDependencies": {
    "vitepress": "^1.6.3",
    "vue": "^3.5.13"
  }
}
```

读法：
| 字段 | 含义 |
|---|---|
| `scripts.dev` | 本地开发：启动预览服务器 |
| `scripts.build` | 构建：把 Markdown 编译成静态 HTML |
| `scripts.preview` | 本地预览构建结果 |
| `devDependencies` | 开发时需要的依赖（VitePress、Vue） |

**你要会的命令（背下来）**：

```bash
npm install      # 根据 package.json 安装依赖 → 生成 node_modules/
npm run dev      # 启动开发服务器，浏览器打开 localhost:5173
npm run build    # 生成最终网站到 docs/.vitepress/dist/
```

**你要学到什么程度**
不需要会写 Node 程序。只要理解：`package.json` 声明依赖 → `npm install` 下载 → `npm run xxx` 执行脚本。

## 四、第 2 层：网站是怎么生成的

### 4.1 VitePress 是什么？

**VitePress** = 基于 Vite + Vue 的**静态站点生成器（SSG）**。

工作流程：

```
你写的 Markdown          VitePress 构建          浏览器看到的
docs/**/*.md      →     编译 + 套主题     →    一堆 .html/.css/.js
config.mts               注入导航侧边栏          完整可访问网站
theme/*                  应用自定义样式
```

「静态」的意思：最终产物是普通 HTML 文件，不需要服务器跑数据库或后端逻辑，所以可以免费托管在 Vercel / GitHub Pages 上。

**为什么选它**（结合本项目）：
- 默认主题已经有导航、侧边栏、搜索、暗色模式、目录
- 用 Markdown 写，学习成本低
- 需要交互时可以嵌 Vue 组件（留言板就是这样）

---

### 4.2 Frontmatter —— 给页面加「元数据」

Markdown 文件顶部用 `---` 包起来的 YAML 区域，叫 frontmatter。VitePress 用它控制页面行为。

**首页**（`docs/index.md`）用了特殊布局：

```markdown
---
layout: home

hero:
  name: "我的技术博客"
  text: "记录学习、思考与成长"
  tagline: "一个计算机专业学生的技术探索空间——Agent 开发、CS 基础、项目实战，写下即是思考"
  actions:
    - theme: brand
      text: 开始阅读
      link: /posts/agent/what-is-agent
    - theme: alt
      text: 关于我
      link: /about

features:
  - icon: 🤖
    title: Agent 开发
    details: AI Agent 的设计思路……
    link: /posts/agent/what-is-agent
---
```

- `layout: home` → 使用首页布局（大标题 + 按钮 + 功能卡片）
- `hero` / `features` → 首页内容的结构化数据，不是手写 HTML

**普通页面**只需要标题：

```markdown
---
title: 关于
---
```

**你要学到什么程度**
会改 `hero`、`features`、`title`；知道 frontmatter 是「页面配置」，正文是「页面内容」。

### 4.3 站点配置 `config.mts` —— 博客的控制台

这是整个站点最重要的配置文件：`docs/.vitepress/config.mts`

```ts
import { defineConfig } from 'vitepress'

export default defineConfig({
  title: '我的技术博客',
  description: 'AI Agent 开发、CS 基础、项目实战……',
  lang: 'zh-CN',

  themeConfig: {
    nav: [ /* 顶部导航 */ ],
    sidebar: { /* 侧边栏，按路径分组 */ },
    socialLinks: [ /* GitHub 图标 */ ],
    footer: { /* 页脚文字 */ },
    search: { provider: 'local' },  // 本地搜索
    outline: { label: '本页目录', level: [2, 3] },
    docFooter: { prev: '上一篇', next: '下一篇' },
  },

  markdown: {
    lineNumbers: true  // 代码块显示行号
  },

  // 加载 Google Fonts（Lora 衬线字体）
  head: [
    ['link', { rel: 'preconnect', href: 'https://fonts.googleapis.com' }],
    ['link', { rel: 'preconnect', href: 'https://fonts.gstatic.com', crossorigin: '' }],
    ['link', { rel: 'stylesheet', href: 'https://fonts.googleapis.com/css2?family=Lora:ital,wght@0,400;0,500;0,600;0,700;1,400;1,500;1,600&display=swap' }]
  ]
})
```

**导航栏 `nav`**（顶部菜单）：

```ts
nav: [
  { text: '首页', link: '/' },
  {
    text: '文章',
    items: [  // 下拉菜单
      { text: 'Agent 开发', link: '/posts/agent/what-is-agent' },
      { text: 'CS 基础', link: '/posts/cs/tcp-handshake' },
      { text: '项目实战', link: '/posts/projects/vitepress-blog' },
    ]
  },
  { text: '留言板', link: '/guestbook' },
  { text: '关于', link: '/about' },
]
```

**侧边栏 `sidebar`**（文章页左侧，按 URL 前缀匹配）：

```ts
sidebar: {
  '/posts/agent/': [
    {
      text: '🤖 Agent 开发',
      items: [
        { text: '什么是 AI Agent？…', link: '/posts/agent/what-is-agent' },
        { text: '提示词工程：…', link: '/posts/agent/prompt-engineering' },
      ]
    }
  ],
  '/posts/cs/': [ /* 同理 */ ],
  '/posts/projects/': [
    {
      text: '🔧 项目实战',
      items: [
        { text: 'VitePress 个人博客从零搭建', link: '/posts/projects/vitepress-blog' },
      ]
    }
  ]
}
```

**`head` 配置项**：`config.mts` 里的 `head` 数组让 VitePress 在页面 `<head>` 里注入标签。本项目用它预连接 Google Fonts 并加载 Lora 字体——标题和部分 UI 文字用 Lora 显示，正文用系统默认中文字体。

**关键认知**：
- 新建一篇文章后，**不会自动出现在侧边栏**，要手动在 `sidebar` 里加一条
- 链接路径对应 `docs/` 下的文件路径（不含 `.md`）
  - 文件：`docs/posts/agent/what-is-agent.md`
  - 链接：`/posts/agent/what-is-agent`

## 五、第 3 层：定制与扩展

### 5.1 CSS 变量与设计系统 —— 从颜色到完整的视觉语言

你已经会 CSS，但这个项目不是"随便写几个颜色"，而是有一套完整的设计系统。

#### 什么是设计系统（Design System）

简单说就是把所有视觉相关的决定打包成**一组有名字的变量**：颜色、字体、间距、圆角不再到处写具体值，而是写变量名。好处：

- 改一个变量 → 全局生效（比如从蓝色调换成绿色调）
- 暗色模式切换自动生效（`.dark` 下换一套变量值）
- 新页面写样式时用现成变量，视觉自动一致

#### 本项目的设计系统："Botanist's Desk"

`docs/.vitepress/theme/custom.css` 定义了一套名为 "Botanist's Desk"（植物学家的书桌）的视觉语言：

| Token | 语义 | 浅色值 | 暗色值 |
|---|---|---|---|
| `--vp-c-brand-1` | 主色（松绿 accent） | `#2c5c3b` | `#5c9a6b` |
| `--vp-c-brand-2` | 辅色（悬停态） | `#3d704c` | `#4a8560` |
| `--vp-c-brand-soft` | 底色（极淡绿） | `rgba(44,92,59,0.08)` | `rgba(92,154,107,0.12)` |
| `--vp-c-bg` | 页面底色（纸色） | `#faf8f5` | `#111311` |
| `--vp-c-bg-soft` | 次级底色（羊皮纸） | `#ede8df` | `#1c1f1c` |
| `--vp-c-text-1` | 主文字（墨色） | `#1a1a1c` | `#e8e4db` |
| `--vp-c-text-2` | 次级文字 | `#5c564e` | `#8b9588` |
| `--vp-c-border` | 边框 | `#d9d3c8` | `#2d332c` |

还有一套字体变量：

```css
--vp-font-family-base:    -apple-system, ...;    /* 正文：系统默认 sans-serif */
--vp-font-family-mono:    'JetBrains Mono', ...;  /* 代码：等宽字体 */
--vp-font-family-heading: 'Lora', 'Noto Serif SC', Georgia, serif;  /* 标题：衬线 */
```

#### 字体策略：为什么标题用 Lora

Lora 是一款有手写笔触的衬线字体，既保留了传统书籍的温润感，又在屏幕上可读。中文环境下降级到系统衬线体。

代码块用 JetBrains Mono——这款字体连字（ligature）表现好，`=>`、`==>`、`!=` 会渲染为单字符造型，编码时更舒适。

#### 暗色模式如何工作

```css
:root {
  --vp-c-brand-1: #2c5c3b;   /* 浅色：深松绿 */
}

.dark {
  --vp-c-brand-1: #5c9a6b;   /* 暗色：亮松绿，背景是深色 */
}
```

VitePress 的暗色切换本质就是把 `:root` 换成 `.dark`，同一套变量名换一套值。你写的覆盖样式只要用变量而不用硬编码颜色，就自动支持暗色。

#### 如何覆盖默认组件

```css
/* 首页标题：Lora 衬线 + 松绿色 */
.VPHero .name {
  font-weight: 600;
  font-size: 3rem;
  color: var(--vp-c-brand-1);
}

/* 功能卡片 hover：左边出现绿色标记线，而不是弹起来 */
.VPFeature {
  border-left: 3px solid transparent;
  transition: border-left-color 0.2s;
}
.VPFeature:hover {
  border-left-color: var(--vp-c-brand-2);
  background: var(--vp-c-bg-soft);
}
```

这种 hover 效果是整个设计的签名——像读书时在段落旁边画一条竖线做笔记，而不是浮起+阴影这种通用套路。

**你要学到什么程度**
- 会改 `:root` / `.dark` 里的颜色变量来换整套配色
- 会用开发者工具审查元素找到 VitePress 的 CSS class
- 理解设计系统 = 有名字的变量 + 一致的用法

### 5.2 一点点 JavaScript —— 只为读懂留言板

你不需要系统学 JS，但要能看懂下面这段在干什么。

`GiscusComment.vue` 的核心逻辑：

```ts
const giscusRef = ref<HTMLDivElement>()

onMounted(() => {
  const script = document.createElement('script')  // 创建一个 <script> 标签
  script.src = 'https://giscus.app/client.js'      // 指向 Giscus 官方脚本
  script.setAttribute('data-repo', 'lfyu778/my-blog')
  script.setAttribute('data-repo-id', 'REPO_ID_HERE')
  // ... 更多 data-* 配置
  script.async = true
  giscusRef.value?.appendChild(script)             // 插入到页面里
})
```

翻译成你熟悉的思路：

```
页面加载完成后：
  1. 动态创建一个 <script> 标签
  2. 设置它的 src 和各种 data- 属性（配置项）
  3. 把它塞进页面的某个 div 里
  → 浏览器加载这个脚本 → Giscus 评论框出现
```

对应的 HTML 部分：

```html
<div ref="giscusRef" class="giscus-container"></div>
```

`ref="giscusRef"` 让 JS 能拿到这个 div，相当于「给这个元素起了个可在脚本里用的名字」。

**你要学到什么程度**
能读懂「创建元素 → 设属性 → 插入 DOM」三步，不要求会独立写复杂逻辑。

### 5.3 Vue 3 组件入门 —— 留言板组件拆解

**Vue** 是前端框架，用来把「结构 + 逻辑 + 样式」打包成可复用的组件。VitePress 底层就是 Vue。

一个 `.vue` 文件通常三块（本项目完整示例）：

```vue
<script setup lang="ts">
// 逻辑：数据、生命周期
import { onMounted, ref } from 'vue'
const giscusRef = ref<HTMLDivElement>()
onMounted(() => { /* 插入 Giscus 脚本 */ })
</script>

<template>
  <!-- 结构：类似 HTML -->
  <div ref="giscusRef" class="giscus-container"></div>
</template>

<style scoped>
/* 样式：scoped 表示只作用于本组件 */
.giscus-container {
  margin-top: 32px;
  min-height: 200px;
}
</style>
```

**主题入口如何注册组件**（`docs/.vitepress/theme/index.ts`）：

```ts
import DefaultTheme from 'vitepress/theme'
import type { Theme } from 'vitepress'
import GiscusComment from './components/GiscusComment.vue'
import './custom.css'

export default {
  extends: DefaultTheme,           // 继承默认主题（导航、侧边栏都还在）
  enhanceApp({ app }) {
    // 注册为全局组件 → 任意 Markdown 里都能写 <GiscusComment />
    app.component('GiscusComment', GiscusComment)
  },
} satisfies Theme
```

所以 `guestbook.md` 里那一行 `<GiscusComment />` 才能生效。

**你要学到什么程度**
- 知道 `.vue` = template + script + style
- 知道 `extends: DefaultTheme` 是「在默认主题上增强」
- 知道 `app.component(...)` 是「注册全局组件」
- 暂时不用会写复杂 Vue 应用

### 5.4 TypeScript —— 能读就行

项目中出现的 TS 特征：

```ts
import type { Theme } from 'vitepress'   // 只导入类型
const giscusRef = ref<HTMLDivElement>() // <HTMLDivElement> 是类型参数
} satisfies Theme                        // 约束对象符合 Theme 形状
```

**你要学到什么程度**
把 `<...>`、`type`、`satisfies` 当成「给编辑器看的注释」，不影响运行逻辑。以后真要写配置再系统学。

## 六、第 4 层：上线与协作

### 6.1 Git + GitHub

**Git**：本地版本管理。每次 `commit` 是一个快照。
**GitHub**：远程仓库托管 + Discussions（留言板依赖它）。

本项目日常循环：

```
改 Markdown / 配置
    → git add .
    → git commit -m "add: 某篇文章"
    → git push
    → Vercel 自动重新部署
```

`.gitignore` 告诉 Git 忽略什么：

```
node_modules/           # 依赖太大，不该进仓库（别人 npm install 即可）
docs/.vitepress/dist/   # 构建产物，线上再生成
docs/.vitepress/cache/  # 缓存
```

---

### 6.2 Vercel 部署

`vercel.json`：

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "docs/.vitepress/dist",
  "framework": "vitepress"
}
```

含义：
1. 收到代码推送后，执行 `npm run build`
2. 把 `docs/.vitepress/dist` 当作网站根目录发布
3. 用 VitePress 相关的默认优化

你不需要自己管服务器、HTTPS、CDN——这些 Vercel 全包了。

---

### 6.3 Giscus 留言系统

**原理**：评论存在你 GitHub 仓库的 Discussions 里，页面通过嵌入 Giscus 脚本显示。

依赖条件：
1. 仓库开启 Discussions
2. 安装 Giscus GitHub App
3. 在 `GiscusComment.vue` 填入真实的 `data-repo-id` 和 `data-category-id`（目前还是占位符）

数据流：

```
访客在网页留言
  → Giscus 脚本
  → 写入 GitHub Discussions
  → 其他人刷新页面就能看到
```

完全免费，不用自己的数据库。

## 七、设计系统深入：如何做出"不模板化"的视觉

这一节是**从本项目真实选择中提炼的设计观念**，适合学会基础后进一步思考。

### 7.1 避开三个默认公式

现在 AI 生成的网页设计集中在三种套路：

1. **暖色学院风**：奶油色底（`#F4F1EA`）+ 衬线标题 + 陶土色 accent
2. **暗色赛博风**：纯黑背景 + 荧光绿或朱红色 accent
3. **报纸排印风**：细线分割 + 零圆角 + 密排多栏

这三种对某些主题本身合理，但不在意主题就用就是默认值，不是选择。

本项目的选择：**松绿 accent + 暖灰羊皮纸底 + Lora 衬线标题**。灵感来自旧教科书里的植物插图——和"做笔记、写学习记录"这个使用场景绑定，不是随便选的。

### 7.2 签名：一个可辨识的特征

「签名」是这份设计里唯一该被记住的东西。

本项目的签名是 **边注式 hover**：Feature 卡片和文章卡片在鼠标悬停时，左边出现一条绿色竖线，像在书页上画线标注。不是常见的浮起（`translateY(-4px)`）+ 阴影扩散。

CSS 实现：

```css
.post-card {
  border-left: 3px solid transparent;  /* 默认透明，看不见 */
  transition: border-left-color 0.2s, background 0.2s;
}
.post-card:hover {
  border-left-color: var(--vp-c-brand-2);  /* hover 时绿色竖线出现 */
  background: var(--vp-c-bg-soft);
}
```

### 7.3 动效策略

这份设计刻意减少了动画。只有边框颜色过渡和背景过渡，没有弹跳、上浮、缩放。理由：一个技术博客不是营销 Landing Page——读者来是为了安静地阅读内容，不是被花哨的动效分散注意力。

唯一保留了 `@media (prefers-reduced-motion: reduce)` 规则，尊重操作系统的减弱动效设置。

## 十一、概念对照速查（HTML/CSS 视角）

| 你已熟悉的 | 本项目对应概念 | 说明 |
|---|---|---|
| `.html` 文件 | `.md` 文件 | 内容载体；构建时变成 HTML |
| `<title>` / meta | frontmatter + `config.mts` 的 title/description | 页面和站点元信息 |
| 手写导航 `<nav>` | `themeConfig.nav` | 配置化，一处修改全局生效 |
| 手写侧边栏 | `themeConfig.sidebar` | 按路径前缀匹配 |
| `style.css` | `custom.css` + 页面内 `<style>` | 用 CSS 变量覆盖主题 |
| CSS 颜色写死 | `--vp-c-brand-1` 等变量 | 一处改，全局换肤；还支持暗色 |
| 复制粘贴的页头页脚 | 默认主题布局 | 你几乎不用写公共布局 |
| `<link>` 加载字体 | `config.mts` 的 `head` 数组 | 在站点配置里声明外部资源 |
| 无 | 设计系统（Design System） | 色彩、字体、间距统一命名，全局一致 |
| 无 | Vue 组件 `.vue` | 把交互封装成标签，Markdown 里直接用 |
| 无 | `package.json` / npm | 管理工具链依赖 |
| 手动 FTP 上传 | `git push` + Vercel | 自动构建部署 |
| 无后端评论 | Giscus + GitHub Discussions | 无服务器的评论方案 |

## 十二、学习时的常见坑

1. **改了文章侧边栏没出现**
   必须在 `config.mts` 的 `sidebar` 里手动加链接。

2. **链接 404**
   路径是 `/posts/agent/what-is-agent`（对 `docs/posts/agent/what-is-agent.md`），不要写 `.md`，也不要写错文件夹名。

3. **`npm run dev` 改了没反应**
   多数情况会热更新；若改的是 `config.mts`，有时需要重启 dev 服务器。

4. **把 `node_modules` 拷给别人**
   不要。让对方自己 `npm install`。这也是 `.gitignore` 忽略它的原因。

5. **留言板空白**
   检查：仓库是否 Public、Discussions 是否开启、`REPO_ID_HERE` 是否已替换、Giscus App 是否安装到该仓库。

6. **改了很久样式没效果**
   检查 CSS 选择器优先级——VitePress 默认样式的优先级可能比你的选择器高。用浏览器开发者工具看哪个规则赢了，把选择器写得更具体一些。

7. **只学工具不写内容**
   技术是为表达服务的。第 1 周就可以开始写真正的笔记，边写边学配置。
